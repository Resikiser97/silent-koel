# 屬性模組化架構設計（v0.2.x 計劃）

## 核心概念

屬性（元素）是可抽離的設計單元，可套用到任何生物上。
套用後生物根據自身類型（近戰/遠戰）獲得對應的技能和效果。

## 屬性定義格式

```
定義 {屬性名稱} {
  [共用] 所有生物都獲得的效果
  [近戰] 僅近戰生物獲得的效果
  [遠戰] 僅遠戰生物獲得的效果
}
```

## 現有屬性定義

### 毒霧屬性
```
定義 毒霧 {
  [共用] 攻擊含毒傷害
  [遠戰] 毒牙攻擊技能（遠程回旋毒牙 projectile）
  [遠戰] 毒霧炮技能（拋射落地範圍毒霧）
}
```
套用範例：
- 毒霧 + 近戰精英 = 毒霧犬（攻擊附帶毒傷）
- 毒霧 + 遠戰精英 = 毒霧隼（毒牙 + 毒霧炮）

### 幽靈屬性
```
定義 幽靈 {
  [共用] 深藍/靛紫外觀，脈衝光環
  [遠戰] Sniper 遠程攻擊（單發直線高速子彈）
  [近戰] 近戰攻擊（標準）
}
```

### 暗影屬性
```
定義 暗影 {
  [共用] 純黑外觀，橘色光邊，旋轉虛線環
  [遠戰] 霰彈槍攻擊（扇形多發散彈）
  [近戰] 近戰攻擊（比幽靈快）
}
```

## Boss 階段屬性

### 狙擊槍屬性
```
定義 狙擊槍 {
  [遠戰] 攻擊距離增加
  [遠戰] 子彈飛行速度增加
  [遠戰] 傷害 +30
}
```

### 霰彈槍屬性
```
定義 霰彈槍 {
  [遠戰] 同時發射 N 顆子彈
  [遠戰] 分子彈呈 Y 角度擴散
}
```

黑色獵人套用範例：
- Phase 1 = 狙擊槍屬性
- Phase 2 = 霰彈槍屬性
- Phase 3 = 狙擊槍屬性 + 霰彈槍屬性

## 設計優勢

1. 新難度只需「組合現有屬性」，不需要重新寫生物邏輯
2. 每次套用後必須 crosscheck：最終 HP / 攻擊力 / 獲得哪些效果
3. 屬性之間互相獨立，避免數據錯誤
