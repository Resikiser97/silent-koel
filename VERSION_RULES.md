# 版本號規則

每次完成修改後必須執行以下步驟：

## 1. 判斷版本號如何更新

版本號格式：`v0.x.y.z`
- `x`：賽季版本（排行榜隔離用），**手動決定何時升**，升了之後排行榜重新競爭
- `y`：功能版本
- `z`：修復版本

更新規則：
- 只修 Bug → `z` +1（例如 v0.0.8.3 → v0.0.8.4）
- 新增功能 → `y` +1，`z` 歸零（例如 v0.0.8.4 → v0.0.9.0）
- 重大重構 → 第一位 +1，後面歸零（例如 v0.0.9.0 → v1.0.0.0）
- 賽季升級 → `x` +1，`y` 和 `z` 歸零（例如 v0.0.65.0 → v0.1.0.0）**由開發者手動執行，不自動觸發**

混合情況（同時有新功能和 Bug 修復）→ 以最高等級決定（新增功能優先於修 Bug）。

## 2. 更新 gameConfig.js

修改 `GAME_INFO.version` 的值為新版本號。

## 3. 在 CHANGELOG.md 最上方新增一條記錄

格式如下：

```
## v0.0.9.0 - 2026-XX-XX

### 新增
- 新功能描述

### 修復
- Bug 修復描述

### 調整
- 數值或行為調整描述
```

只填有內容的小節，沒有的小節省略不寫。

## 4. 更新 patchnotes.js

在 `PATCH_NOTES` 陣列最上方新增本次版本公告（格式參考現有條目）。

## 5. 更新 MAIN.md 和 project_summary.md

若本次修改涉及新函式、新模組、或重要設計變更，同步更新對應說明。

## 注意事項

- 每次對話結束前必須完成以上步驟，不需要等用戶提醒
- 日期使用 YYYY-MM-DD 格式，取當前對話日期
- CHANGELOG.md 最新版本永遠在最上方
- `x`（賽季版本）**只有開發者明確指示才能升級**，Claude Code 不可自行決定
