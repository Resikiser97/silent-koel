# 版本號規則

每次完成修改後必須執行以下步驟：

## 1. 判斷版本號如何更新

- 只修 Bug → 最後一位 +1（例如 v0.8.3 → v0.8.4）
- 新增功能 → 中間數字 +1，最後歸零（例如 v0.8.4 → v0.9.0）
- 重大重構 → 第一位 +1，後面歸零（例如 v0.9.0 → v1.0.0）

混合情況（同時有新功能和 Bug 修復）→ 以最高等級決定（新增功能優先於修 Bug）。

## 2. 更新 gameConfig.js

修改 `GAME_INFO.version` 的值為新版本號。

## 3. 在 CHANGELOG.md 最上方新增一條記錄

格式如下：

```
## v0.8.4 - 2026-XX-XX

### 新增
- 新功能描述

### 修復
- Bug 修復描述

### 調整
- 數值或行為調整描述
```

只填有內容的小節，沒有的小節省略不寫。

## 注意事項

- 每次對話結束前必須完成以上三個步驟，不需要等用戶提醒
- 日期使用 YYYY-MM-DD 格式，取當前對話日期
- CHANGELOG.md 最新版本永遠在最上方
